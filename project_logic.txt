Project Logic Explanation

1. Program startup
- The program creates three main objects: a Blueprint, a Funcs helper, and a Player.
- The player is named "William" and starts in room 0.
- The game then prints the player's location and available paths.

2. Blueprint and map
- The Blueprint class builds an array of Room objects.
- Each Room has:
  - Id: unique number for the room.
  - Name: room name.
  - Description: room description.
  - avaliableRooms: list of room ids the player can move to from this room.
  - IsLocked: whether the room is locked and cannot be entered.
- This is the project map: rooms are connected by ids instead of coordinates.

3. WhereAmI method
- Funcs.WhereAmI finds the room that matches the player's CurrentRoomId.
- It prints the room name and description.
- Then it calls WhereCanIGo to show where the player can move next.

4. WhereCanIGo method
- It lists all connected rooms from the current room's avaliableRooms.
- The program asks the user to enter a number to choose a destination.
- The chosen number selects one of the available room ids.
- Then it calls Move() with the selected room id.

5. Move method
- Move finds the destination room by id.
- If the room is locked, it prints a message and returns the player to the current room view.
- If the room is unlocked, it updates player.CurrentRoomId.
- It clears the console and shows the new room with WhereAmI again.

6. Inventory and items
- The Player class has an ItemArray for inventory, but the main program does not use it yet.
- Funcs.AddItemToInventory can add an item to the first empty slot.
- Funcs.ItemInRoom can place a "Key" item if the player is in room 7, but that logic is not currently used by the main game flow.

Summary
- The project is a basic text adventure map explorer.
- The player moves between rooms by choosing one of the listed connected rooms.
- Locked rooms block movement until they are unlocked or bypassed in future logic.
